Jenkins Pipeline Configuration

Our Jenkins pipeline automates the entire deployment process with multiple stages.

pipeline {
    agent any
    stages {
        stage('Compile') {
            steps {
                sh 'mvn clean compile'
            }
        }
        stage('Code Review') {
            steps {
                sh 'mvn checkstyle:checkstyle'
            }
        }
        stage('Unit Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Package') {
            steps {
                sh 'mvn package'
            }
        }
        stage('Deploy') {
            steps {
                sshagent(['tomcat-server']) {
                    sh 'scp target/*.war ubuntu@ec2-ip:/opt/tomcat/webapps/'
                    sh 'ssh ubuntu@ec2-ip "sudo systemctl restart tomcat"'
                }
            }
        }
    }
}

Deployment Pipeline Workflow

Our automated pipeline follows these steps to ensure seamless deployment from GitHub commit to production on Tomcat.

1. Code Push to GitHub

Developer pushes Java code changes to the GitHub repository, triggering the automated pipeline.

2. GitHub Webhook Trigger

GitHub webhook automatically notifies Jenkins server about the new commit, starting the build process.

3. Maven Build & Test

Jenkins runs Maven commands to compile, run unit tests, perform code review, and package the application into a WAR file.

4. Deploy to Tomcat on AWS EC2

Jenkins uses SSH to copy the WAR file to AWS EC2 Tomcat server and restarts the service for immediate deployment.
